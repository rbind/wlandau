workflowHelper

This package is for anyone who uses R to analyze multiple datasets in multiple ways. With minimal code, you can smoothly can generate datasets, apply multiple methods of analysis to each dataset, summarize each analysis of each dataset, collect the summaries, and generate presentable output. And the best part is that workflowHelper is built on top of Rich Fitzjohn's remake package, so when you change your code, only the affected pieces of your workflow will be recomputed when you run your workflow again. Remake is smart, so meaningful changes to code trigger rebuilds, but changes to comments and whitespace do not. Deploy or update your workflow with a single call to GNU Make, and take advantage of the parallel computing available via parallelRemake.

parallelRemake

This R package is a slight extension of Rich Fitzjohn's remake. Remake does not support parallel computing natively, so I wrote parallelRemake in order to generate a Makefile that distributes remake rules over simultaneous processes. ParallelRemake is also a building block of workflowHelper.

fbseq

This R package toolkit is at the center of my dissertation research. It fits a hierarchical model to RNA-sequencing data and other high-dimensional count data in fully Bayesian fashion via Markov chain Monte Carlo (MCMC). I accelerated the MCMC using massively parallel, the machinery for which is encapsulated in separate helper packages. The fbseqCUDA package implements GPU-accelerated MCMC with tens of thousands of simultaneous threads, and the fbseqOpenMP implements the same parallelism on a smaller scale using OpenMP. Publication of the method is pending.
