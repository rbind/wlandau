Original post

Personal accounting can be frustrating when there are a lot delayed transactions. With pending charges and charges that take some time to even reach the bank, it can be hard to know how much you actually have left. That's why I created Dynamic Ledger.

The program accepts a tab-delimited ledger file like the one below as input.

This is not my personal ledger, of course, but it is a useful example. If the file above is called ledger.txt, then I can open Terminal (or another command line interface program) and type 
dl ledger.txt
And I get a summary of my ledger.



The summary tells me that I have $0.00 "available" on my credit card and $1,335.00 "available" in my checking account. These are the balances that I should see on my banks' websites and mint.com if I've done my bookkeeping correctly. But in reality, I owe my credit card company $45.50 and I only have $1,289.50 of real money in my bank account because of the two delayed transactions at the top of the ledger file.

Currently, the program is an open source Linux/Unix-style package written in C. You can clone it from my GitHub page or download the source as a tarball from here. It should be compatible with your platform as long as you have a C compiler. Detailed installation and usage instructions are explained in the man page and pdf tutorial.
